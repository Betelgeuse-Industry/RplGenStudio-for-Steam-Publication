# 概述
这是一基于python3 和pygame2.1 的自动replay生成工具，旨在降低replay视频制作中的重复工作，节约时间提升效率。<br>
包依赖：pygame pandas numpy

# 快速上手
1. 下载源代码和包含了示例文件（./toy/）的压缩包并解压
2. 在终端运行代码，即可开始放映示例项目。<br>
`python replay_generator_en_alpha1.1.py -l 3.sound_test.txt -d 1.MediaObject.txt -t 2.CharactorTable.csv`
3. 进入程序后，按空格键（space）开始播放，播放的过程中，按a键跳转到前一小节，d键跳转到后一小节，Ese键终止播放并退出

# 参考文档

## 程序参数
1. --LogFile, -l ：log文件的路径，文件格式要求详见 文件格式.log文件
2. --MediaObjDefine, -d ：媒体定义文件的路径，文件格式要求参考 文件格式.媒体定义文件
3. --CharacterTable, -t ：角色表文件的路径，格式为制表符分隔的数据表，包含至少Name、Subtype、Animation、Bubble4列。
4. --OutputPath, -o ：可选的参数，输出文件的目录；如果输入了该标志，则项目的时间轴和断点文件将输出到指定点目录，格式为.pkl，是pandas.DataFrame 格式，可以在python中使用pd.read_pickle()函数读取。
5. --FramePerSecond -F ：可选的参数，播放的帧率，单位是fps；默认值是30fps。
6. --Width –W ：可选的参数，窗体的宽；默认值是1920。
7. --Height -H ：可选的参数，窗体的高；默认值是1080。
8. --Zorder –Z ： 可选的参数，渲染的图层顺序；通常不建议修改这个参数，除非必要。格式要求详见 进阶使用.图层顺序

## 1. 媒体定义文件
媒体定义文件是定义每个媒体的信息，直接使用的python代码进行定义。
目前版本可用的媒体对象包括：
1.	文本 Text(fontfile='C:/Windows/Fonts/simhei.ttf',fontsize=40,color=(0,0,0,255),line_limit=20)
	- 文本是气泡对象的一部分，不可使用
	- fontfile 参数需要指定一个字体文件的路径，默认参数是系统的微软雅黑字体。
	- fontsize 参数设置了字体的字号，合理的参数是大于0的整数
	- color 参数设置了字体的颜色，是一个有4个元素的tuple，对应(R,G,B,A)，默认值是黑色；四个元素应为0-255的整数。
	- line_limit 参数设置了单行显示的字符数量的上限，默认为20字；超过上限会触发自动换行。
	> 注意：因为气泡需要引用文本对象，因此，文本对象的定义必须在气泡对象的定义之前。
2.	气泡 Bubble(filepath,Main_Text=Text(),Header_Text=None,pos=(0,0),mt_pos=(0,0),ht_pos=(0,0),line_distance=1.5)
	- 气泡是一个文本框，在角色发言时显示，包含了主文本、头文本、底图三个组成部分。
	- filepath 参数需要指定一个图片文件的路径；即使不需要底图，也需要指定一个空白底图的路径
	- Main_Text 参数需要为主文本指定一个文本对象，主文本对应log文件中的发言文本
	- Header_Text 参数需要为头文本指定一个文本对象，默认为None，既无头文本。头文本对应发言者的角色名。
	- pos 参数指定了气泡在屏幕上的位置，是一个2元素的tuple，对应(X,Y)，默认为(0,0)，即左上角。
	- mt_pos 参数指定了主文本相对于气泡底图的位置，是一个2元素的tuple，对应(X,Y)，默认为(0,0)，即左上角。
	- ht_pos 参数指定了头文本相对于气泡底图的位置，是一个2元素的tuple，对应(X,Y)，默认为(0,0)，即左上角。
	- line_distance 参数设置了多行显示时的行距，默认值为1.5倍行距
3.	背景 Background(filepath,pos = (0,0))
	- 背景指整个屏幕的背景，通常位于最下的图层，可以在log文件中的背景行中设置背景及其切换效果
	- filepath 参数需要指定一个图片文件的路径；或者，指定{'black','white','greenscreen'}中的一个。
	- pos 参数指定了背景在屏幕上的位置，是一个2元素的tuple，对应(X,Y)，默认为(0,0)，即左上角。
	> 注意：由于背景图通常都是全屏的图片，因此不建议修改Background的pos的默认值。
4.	立绘 Animation(filepath,pos = (0,0))
	- 立绘指和角色绑定的个人形象图片，通常位于背景的上层，气泡的下层。
	- 一个角色可以在不同的subtype下指定不同的立绘，用于实现差分效果；使用时在log文件的对话行里指定到不同的subtype。
	- filepath 参数需要指定一个图片文件的路径。
	- pos 参数指定了立绘在屏幕上的位置，是一个2元素的tuple，对应(X,Y)，默认为(0,0)，即左上角。
	> 注意：如果希望实现多人同框效果，建议为同框时的立绘额外定义一个Animation对象，并指定合适的位置。
5.	背景音乐 BGM(filepath,volume=100,loop=True)
	- BGM指长的，一直位于后台循环播放的音频。
	- 支持的格式是.ogg，如果是.mp3格式的背景音乐，建议先进行格式转换
	- filepath 参数需要是一个音频文件的路径，格式为字符串，需要引号引起来
	- volume 参数指定背景音乐的音量，合理的参数是0-100的整数
	- loop 参数决定背景音乐是否会循环播放；如果否，设置为False
	> 注意：BGM建议使用.ogg格式的音频，否则有可能出现程序的不稳定。另外，建议在后期制作软件中手动加入BGM。
	> 注意：BGM和audio的逻辑不同，不可混用！
6.	音效 Audio(filepath)
	- 音效，指播放的短音频，短音频通常只会完整地播放一次
	- filepath 参数需要是一个音频文件的路径，格式为字符串，需要引号引起来
	> 注意：replay视频中通常包含大量的语音文件，不建议均定义为Audio对象，会消耗较大的内存，在Log文件的对话行的音效框里指定文件路径即可。

## 2. 角色设置文件
- 角色设置文件是一个制表符分隔的数据表文件，用来指明每个角色的每一张立绘，对应的对话气泡和立绘图片。
- 需要包括Name、Subtype、Animation、Bubble四个列
- 每个name需要至少有一个subtype是default，Animation和Bubble需要指向相应的媒体对象。
例子：

|Name|Subtype|Animation|Bubble|
|:---|:---:|:---:|:---:|
|张安翔|default|zhang|bubble1|
|我|default|drink|bubble2|

> 注意：这个设置文件不仅包括角色设置，实际上，骰子，旁白都可以以“角色”的形式指定。

## 3. Log文件
log文件是整个演示的主文件，决定了展示的内容和效果。log文件有3种有效行，对话行，背景行和设置行，分别有其对应的格式。在下面的介绍中，格式里的各个部分将由不同的颜色展示。
### A. 对话行
`[name1(100).default,name2(60).default,name3(60).default]<replace=0>:abandon#hostile.<all=0>{"./audio/1.ogg";30}{Audio;30}`
1.	角色框：\[name(alpha).subtype;...\]
	- 角色框内最多选择3个角色，同框角色的立绘都将展示出来。
	- 只有顺位第一个角色的立绘的透明度为100，其余角色会自动被设置为半透明，在角色名后使用(alpha)可以手动指定立绘的透明度。
	- 同一个角色如果存在差分立绘，可以在角色名后使用.subtype来指定差分；未指定的将默认使用.default的立绘。
2.	切换效果修饰符：\<method=time\>
	- 目前可用的切换效果有replace和black，若语句中未指定切换效果，则默认为\<replace=0\>
	- 可以缺省持续时长，此时持续时长将参考
	- Replace：直接替换，持续时长指切换延后时间
	- Black：黑场，以渐变为切换，持续时长指渐变点时长
3.	发言文本：talk#
	- 发言文本可以是大部分文本，但不建议包括英文方括号“[]”、英文尖括号“<>”和英文花括号“{}”，否则可能导致程序的不稳定或报错。
	- 发言文本中可以使用井号作为手动换行符。
	- 需要注意的是，在手动指定换行符的行内，自动换行是失效。
4.	文本效果修饰符：\<method=time\>
	- 文本展示的效果有all，w2w，l2l，默认为\<all=0\>
	- all，一次性展示所有文本，参数设置展示的延迟时间
	- w2w，逐字展示文本，参数为展示速度，单位为字每秒
	- l2l，逐行展示文本，参数为展示速度，单位为行每秒
5.	音效框{file_or_obj;time}
	- 音效可以指定一个Audio对象，或者一个文件的路径,文件的路径必须用引号引起来
	- 延迟时间只这个音效相对于本小节第一帧所延迟的帧数
	- 一个对话行可以有多个音效框

### B. 背景行
`<background><replace=0>:Background`
1.	\<background\> 是背景行的识别标志 ，是背景行的必要组成部分。
2.	\<切换效果修饰符\> 背景展示修饰方法有：
	- cover：覆盖，新的背景会逐渐覆盖原背景，参数是整个渐变的时长
	- black，黑场，原背景先隐入黑场，然后新背景再逐渐出现，参数是整个渐变的时长。
	- white，白场，原背景先隐入白场，然后新背景再逐渐出现，参数是整个渐变的时长。
	- replace，替换，瞬间替换，参数是替换发生的延迟时间。默认值是replace=0

### C. 设置行
`<set:method_default>:<replace=0>`
set:后跟需要设置的全局变量名，可以通过set动态修改的全局变量有：
1.	method_default：默认展示方法，当初始值是\<replace=0\>
	- 当对话行和背景行中缺省\<切换效果修饰符\>时，则使用该默认值
	- 不建议将该默认值设置为仅背景行可用的修饰符，例如cover
2.	text_method_default：默认文本展示方法，初始值是\<all=1\>
	- 当对话行中缺省<文本效果修饰符>时，使用该默认值
	- 例如[name]:talk，等价于[name]\<replace=0\>:talk\<all=1\>
3.	method_dur_default：默认展示时间，初始值是10
	- 当对话行和背景行的<切换效果修饰符>中未指定时间，则使用该默认值
	- 例如 \<replace\>，等价于\<replace=10\>
4.	text_dur_default：默认文本展示时间，初始值是8
	- 当对话行的<文本效果修饰符>中未指定时间，则使用该默认值
	- 例如 talk\<l2l\>，等价于talk\<l2l=8\>
5.	speech_speed：语速，初始值是220
	- 语速主要影响每个小节的持续时间，需要调整语速和语音文件相一致
6.	BGM背景音乐
	- 使用\<set:BGM\>: 设置背景音乐时，需要指定一个BGM对象，或一个.ogg音频文件的路径
> 注意：使用非.ogg文件作为背景音乐，可能导致程序的不稳定，或者卡死！

### D. 注释行、空白行
`#annotation`
1.	当一个行的第一个字符是井号“#”，则这个行被认作为注释，行内的任何内容都不会被执行
2.	log文件可以任意地添加空白行，且不会影响程序的正常使用
